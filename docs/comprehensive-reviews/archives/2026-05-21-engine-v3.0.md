## IN-2 Fix Detail

**Files:** [`src/engine/risk_profile.h`](../../src/engine/risk_profile.h), [`src/engine/risk_profile.cpp`](../../src/engine/risk_profile.cpp), [`tests/test_risk_profile.cpp`](../../tests/test_risk_profile.cpp)

The `RiskProfile` struct is now a standalone typed module (not embedded inside `RiskController`). Its C++ defaults at [`risk_profile.h:14-32`](../../src/engine/risk_profile.h):

```cpp
double riskPct{0.01};
double slMultiplier{1.5};
double maxPositionNotionalXAvailableBalance{0.5};
double maxTotalNotionalPct{8.0};
double softMaxDrawdown{0.20};
double hardMaxDrawdown{0.35};
double softMinUpi{0.5};
double hardMinUpi{-1.0};
double softLimitNetBeta{0.5};
double hardLimitNetBeta{1.0};
double maxGrossBeta{3.0};
```

These **exactly match** the `"moderate"` block in `config.json`. The drift flagged in v1.0 is gone: C++ and JSON are now in sync with a single authoritative source (the header).

`RiskProfileLoadResult::loadActive()` ([`risk_profile.cpp:35-84`](../../src/engine/risk_profile.cpp)) handles all loading paths correctly:

- No `active_profile` key → disabled (legacy mode, existing config fields authoritative)
- Non-string or unknown `active_profile` → falls back to `moderate` with a warning
- Valid named profile → loads all fields with `nlohmann::json::value()` default chaining (missing fields fall back to `moderate`)
- Invalid profile (fails `validate()`) → falls back to `moderate` with a warning

`validate()` ([`risk_profile.cpp:7-33`](../../src/engine/risk_profile.cpp)) catches all invariant violations (range, ordering, sign constraints) and throws `std::invalid_argument`, keeping the loader's catch path safe.

**New test coverage — `tests/test_risk_profile.cpp` (154 lines):**
Covers: disabled (no active_profile key), unknown-profile fallback + warning text, full valid load with all fields, invalid-value fallback + validation error message, warning accumulation. The prior gap of "no test for RiskProfile loading logic" is closed.
