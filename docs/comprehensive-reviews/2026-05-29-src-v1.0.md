## Overview

This whole-tree pass found systemic production blockers across transport, REST signing/rate limiting, order protection, engine reconciliation, backtest/live-signal parity, and orchestration model promotion. Several issues can create unprotected live positions, accept stale or unauthenticated market data paths, undercount Binance rate limits, or make risk gates fail open on malformed data. `ws/` and root-level files were reviewed locally because the final `ws` subagent hit the runtime usage limit; no module was silently dropped.

| Dimension | Assessment |
|---|---|
| Code quality | Large portions are modular, but safety invariants are split across layers and several APIs have unsafe defaults. |
| Architecture | Strategy/catalog/plugin boundaries and live orchestration need stronger lifetime, reload, and fail-closed contracts. |
| Security / data integrity | TLS hostname verification, plugin loading, child-process environments, raw signatures, and secret-copying need hardening. |
| Performance | REST/SQLite/journal paths contain blocking operations on trading executors and undercount Binance endpoint costs. |
| Concurrency | Multiple critical races exist in WebSocket transport, scanner callbacks, parser state, registry reload, detached autotune, and shutdown paths. |
| Testing | Existing module tests are useful, but missing cross-layer tests for order protection, fill reconciliation, rate-limit costs, stale data, and risk fail-closed behavior. |
