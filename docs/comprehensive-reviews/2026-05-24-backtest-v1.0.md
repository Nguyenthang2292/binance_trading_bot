# Code Review: `src/backtest` module

**Date:** 2026-05-24
**Reviewer:** Claude (Opus 4.7)
**Scope:** All files under `src/backtest/`
**Branch:** main

---

## Summary

The module implements a single-direction backtest gate that does walk-forward, anchored-expanding grid search over Gemini-proposed parameter ranges, then validates via plateau finding + majority vote. The design is documented and the trading semantics are conservative, but there are several real bugs (one logic dead-code, one potential Windows subprocess deadlock, an unbounded cache, FP-equality on parameter grids) and notable structural concerns (a 327-line orchestration method, unused config knobs).

## Critical Issues

| # | File | Line | Issue | Severity |
|---|------|------|-------|----------|
| 1 | `parameter_space.cpp` | 191 | `clampToBudget` "no progress" check is dead code — condition `totalCombos <= maxTotalCombos` is provably false inside the `while (totalCombos > maxTotalCombos)` loop. When step widening yields no improvement (e.g., every range already collapsed to 1 step), the loop will spin up to 64 iterations bloating step values absurdly instead of bailing out. Fix: `if (newTotal >= totalCombos) { ... revert and break; }` | 🔴 Critical |
| 2 | `gemini_range_proposer.cpp` | 358–411 (Windows) | Subprocess's **stdout pipe is created but never read** (`outR` is held, never drained). If the Python module writes anything to stdout that exceeds the pipe buffer (~64 KB on Windows), `WriteFile` in the child blocks and the parent stalls until the timeout fires. Drain `outR` alongside `errR`, or redirect stdout to NUL. | 🔴 Critical |
| 3 | `gemini_range_proposer.cpp` | 245–251 | `putCached` only evicts *expired* entries when at capacity. If all `maxEntries` items are still live, the cache **grows unbounded** — `m_cache[key] = ...` insert proceeds regardless. Add LRU/oldest-expiry eviction when expired sweep frees nothing. | 🔴 Critical |
| 4 | `backtest_engine.cpp` | 158–168 | Tick-rounding direction is asymmetric: SL rounds **toward entry** (Long Up / Short Down) → loss optimistically smaller; TP also rounds **toward entry** (Long Down / Short Up) → profit conservatively smaller. Combined, this is a structural bias: backtest underestimates both edges, not symmetric. Verify the live executor rounds identically; otherwise PnL doesn't match. The header claims "tick-size SL/TP quantization" without specifying direction — please document and unit-test against the live path. | 🟡 High |

## What Looks Good

- **Walk-forward math is sound**: anchored-expanding splits via `B = isFraction/(1-isFraction)` give clean, non-overlapping IS/OOS slices and correctly carve from the END so stale prefix is dropped.
- **Backtest semantics are explicit and conservative**: next-bar entry, SL-wins on same-candle collision, leveraged round-trip fees, slippage always against the trader.
- **Strong separation of concerns at the seams**: `IRangeProposer`, `IHistoricalWindowProvider`, `IOptimizableStrategy` are clean ports; `FakeRangeProposer` enables hermetic tests.
- **Structured event logging** at every gate phase (`BACKTEST_GATE_ENTER`, `_DATA_READY`, `_GEMINI_OK`, `_GRID_BUILT`, `_FILTER_DONE`, `_PLATEAU`, `_VOTE`, `_PASS`/`_DROP`) gives good post-mortem traceability.
- **Subprocess timeout + deadline propagation** is correctly threaded from the controller down to the Python module via `remainSecs - 1`.
- **Plateau revalidation** of constraints (`plateau_finder.cpp:218–231`) with fallback to best point is a nice defensive touch.
- The `optimizable_strategy.h` header explicitly documents the v1.1 limitation that plugin-override params aren't threaded through — honest docs.

---

## Verdict

**Request Changes.** Critical issues #1–#3 are real bugs that should be fixed before any production reliance. #4 (rounding asymmetry) needs a clear answer about parity with the live executor — either match it or document the divergence. The rest are quality-of-life and should land before the next milestone.

---

## Recommended Fix Order

1. **`parameter_space.cpp:191`** — one-line fix; remove the dead conjunct from the early-termination check.
2. **`gemini_range_proposer.cpp` Windows stdout drain** — drain `outR` symmetrically to `errR` in the wait loop, or set `si.hStdOutput = devNull`.
3. **`gemini_range_proposer.cpp:247–251`** — fall back to evicting oldest-by-`expiresAt` (or insertion order) when the expired sweep finds nothing.
4. **`backtest_engine.cpp:158–168`** — write a parity test against the live executor's SL/TP rounding before deciding the fix.
5. The rest of the suggestions can be batched into a follow-up cleanup PR.
